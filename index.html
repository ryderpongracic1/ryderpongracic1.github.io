Featured · Apr–May 2026 Go · gRPC · protobuf · LSM-Tree · Raft · Docker

distrikv — Distributed Key-Value Store ↗

A production-quality, strictly consistent CP system — hand-rolled in Go.
  • Consistent Hash Ring — each physical node holds 150 virtual positions on a uint32 ring; keys are placed by hashing MD5(key)[0:4] and walking clockwise. GetN(key, R) returns R distinct physical nodes, explicitly skipping duplicate physical hosts — the naive "next N vnodes" approach fails here and produces hot replicas.
  • LSM-Tree storage engine — writes land in a sorted, RWMutex-protected MemTable (btree.BTreeG), freeze into an immutable buffer, and flush to SSTables on disk. Bloom filters (FNV-1a) skip SSTables on point reads, bounding read amplification; background compaction merges levels to keep read costs in check.
  • Write-Ahead Logging — CRC32 framing + fsync on every append; replay logic handles truncated entries safely on crash recovery. Atomic state persistence uses write-to-temp + Sync + os.Rename — no partial files, ever.
  • Raft consensus — full implementation with a pre-vote phase and heartbeat mechanism for resilient leader election; InstallSnapshot transfers binary state to lagging followers over gRPC. Ships with a first-class distrikv-cli (Cobra, injected AppContext, zero global state) and Docker Compose cluster.
Jan–May 2026Go · SQL · Columnar Storage

vexq — Vectorized SQL Query Engine ↗

A complete analytical query pipeline, written from scratch in Go.
  • Custom .vxq format — file header → row groups (65,536 rows) → footer; 1,024-row blocks with 128-byte null bitmap + typed payload + CRC32. Strings are always dictionary-encoded per row group, reducing equality checks to integer comparison in the filter hot path. Zone maps store per-row-group min/max/nullcount so entire row groups are skipped before any block I/O.
  • Vectorized Volcano engine — Filter uses selection vectors with zero allocations on the hot path; Pratt SQL parser (recursive-descent) feeds a predicate-pushdown + column-pruning optimizer. Result: 4.5× over SQLite on TPC-H Q1/Q3/Q6/Q12 at 6M rows.4.5× SQLite
  • Morsel-driven parallelism — row groups fan out across goroutines, merged by a lock-free partial HashAggregate — an additional 2.5× on aggregate queries.+2.5× on agg
