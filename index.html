§ 03 Selected projects Five works
Featured · Apr–May 2026 Go · gRPC · LSM-Tree · Raft · Docker

distrikv — Distributed Key-Value Store ↗

A production-quality, strictly consistent CP system — hand-rolled in Go.
  • Consistent Hash Ring — 150 virtual positions per physical node on a uint32 ring
    • Keys placed via MD5(key)[0:4] → uint32, walk clockwise to nearest vnode
    • GetN(key, R) returns R distinct physical nodes — explicitly skips duplicate physical hosts
  • LSM-Tree storage engine
    • Writes: MemTable (sorted btree.BTreeG, RWMutex) → immutable buffer → SSTable flush
    • Bloom filters (FNV-1a) skip SSTables on point reads; background compaction merges levels to bound read amplification
  • Write-Ahead Logging — CRC32 framing + fsync on every append
    • Truncated-entry replay-safe on crash recovery
    • Atomic state persistence: write-to-temp + Sync + os.Rename — no partial files, ever
  • Raft consensus — pre-vote phase + heartbeat for resilient leader election
    • InstallSnapshot transfers binary state to lagging followers over gRPC
    • Ships with distrikv-cli (Cobra, injected AppContext, zero global state) and Docker Compose cluster
Jan–May 2026Go · SQL · Columnar Storage

vexq — Vectorized SQL Query Engine ↗

A complete analytical query pipeline, written from scratch in Go.
  • Custom .vxq columnar format
    • File header → row groups (65,536 rows) → footer; 1,024-row blocks with null bitmap + typed payload + CRC32
    • Strings always dictionary-encoded per row group — equality checks become integer comparison in the filter hot path
    • Zone maps (min/max/nullcount) skip entire row groups before any block I/O
  • Vectorized Volcano execution engine — Filter uses selection vectors, zero allocations on the hot path
    • Pratt SQL parser (recursive-descent) → predicate-pushdown + column-pruning optimizer
    • 4.5× over SQLite on TPC-H Q1/Q3/Q6/Q12 at 6M rows4.5× SQLite
  • Morsel-driven parallelism — row groups fan out across goroutines
    • Lock-free partial HashAggregate merge — additional 2.5× on aggregate queries+2.5× on agg
